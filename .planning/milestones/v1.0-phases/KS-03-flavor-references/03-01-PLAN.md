**## Challenge / remove item**
Two sub-workflows:

Remove item (numbered steps): (1) Fetch clearing MetaEvidence (ID 1, → shared-metaevidence.md § LGTCR specifics), (2) Read removal policy, (3) Draft removal evidence JSON + upload to IPFS (→ shared-ipfs-upload.md), (4) Compute removal deposit (removalBaseDeposit() + arbitrationCost, → shared-deposits.md § LGTCR specifics), (5) Simulate, (6) Call `removeItem(itemID, "/ipfs/<CID>")`. ABI: `function removeItem(bytes32 _itemID, string _evidence) external payable`.

Challenge item (numbered steps): (1) Determine request type — check whether the active request is a registration (submission) or removal request; this determines which deposit to use, (2) Read the applicable policy (registration or clearing depending on what's being challenged), (3) Draft challenge evidence JSON + upload to IPFS, (4) Compute challenge deposit: if registration request → `submissionChallengeBaseDeposit() + arbitrationCost`; if removal request → `removalChallengeBaseDeposit() + arbitrationCost` (→ shared-deposits.md § LGTCR specifics), (5) Simulate, (6) Call `challengeRequest(itemID, "/ipfs/<CID>")`. ABI: `function challengeRequest(bytes32 _itemID, string _evidence) external payable`.

**## Submit evidence**
2-step workflow: (1) Draft evidence JSON + upload to IPFS (→ shared-ipfs-upload.md), (2) Call `submitEvidence(itemID, "/ipfs/<CID>")`. ABI: `function submitEvidence(bytes32 _itemID, string _evidence) external`. Note: evidence can be submitted by any party while a request is active.

**## Fund an appeal**
This section MUST include the full step-by-step algorithm (fulfilling handoff from shared-deposits.md:112). Structure:

For the appeal funding formula: `shared-deposits.md § Appeal funding (shared)`.

Stop conditions — do NOT attempt to fund if: no active dispute (disputed == false), loser has already passed the half-time deadline ((start + end) / 2), side is already fully funded (hasPaid[side] == true), simulation fails.

Step-by-step algorithm:
1. Read `getItemInfo(itemID)` → `numberOfRequests`. Set `requestID = numberOfRequests - 1`.
2. Read `getRequestInfo(itemID, requestID)` → `disputed`, `disputeID`, `numberOfRounds`, `ruling`, `requestArbitrator`, `requestArbitratorExtraData`. If `disputed == false`: no appeal to fund — stop.
3. Set `roundID = numberOfRounds - 1`.
4. Read `getRoundInfo(itemID, requestID, roundID)` → `amountPaid[3]`, `hasPaid[3]`.
5. From `requestArbitrator` (NOT the registry-level arbitrator — use the one returned by getRequestInfo): call `appealCost(disputeID, requestArbitratorExtraData)`. Optionally: `currentRuling(disputeID)` and `appealPeriod(disputeID)` → `(start, end)`.
6. Compute `totalRequired` per the formula in `shared-deposits.md § Appeal funding (shared)`.
7. `remaining = hasPaid[side] ? 0 : max(totalRequired - amountPaid[side], 0)`.
8. Simulate then call `fundAppeal(itemID, side)` with `msg.value = remaining`.
Party enum: 0 = None, 1 = Requester, 2 = Challenger. Never fund side 0 (None).
Loser half-time rule: loser must fund before `(start + end) / 2`.

**## Execute / Withdraw**
Brief 2-line section: after the challenge window expires with no challenge, call `executeRequest(itemID)` to finalize. To withdraw fees/rewards after a resolved dispute: `withdrawFeesAndRewards(beneficiary, itemID, requestID, roundID)`. ABI: `function executeRequest(bytes32 _itemID) external` and `function withdrawFeesAndRewards(address _beneficiary, bytes32 _itemID, uint256 _requestID, uint256 _roundID) external` — see shared-abi-fragments.md.
