Key formulas to include:

Hard rules (shared, no subsection):
  - Never quote "typical" deposits — WHY: amounts are governance parameters, not constants
  - UI trap: Curate UI shows base deposit only; true msg.value = base + arbitration cost — WHY: a reverted transaction loses gas
  - Always live-read; optional cross-check via recent successful tx inspection (not primary)

Arbitration cost read (shared, no subsection):
  1. Call registry.arbitrator() → arbitratorAddress
  2. Call registry.arbitratorExtraData() → extraDataBytes
  3. Call arbitrator.arbitrationCost(extraDataBytes) → costInWei
  (pointer): "(full IArbitrator signature: shared-abi-fragments.md)"

LGTCR formulas (### LGTCR specifics):
  Submission:  msg.value = submissionBaseDeposit() + arbitrationCost
  Removal:     msg.value = removalBaseDeposit() + arbitrationCost
  Challenge (registration request): msg.value = submissionChallengeBaseDeposit() + arbitrationCost
  Challenge (removal request):      msg.value = removalChallengeBaseDeposit() + arbitrationCost
  Currency: native token (ETH on Mainnet/Sepolia, xDAI on Gnosis)
  D-02 Scout pointer: "(Scout registries use these same LGTCR deposit formulas — see scout-registries.md for Scout-specific context.)"

PGTCR specifics (### PGTCR specifics):
  Two-asset model WHY: ERC20 permanent stake = collateral (stays if item removed); native cost = arbitrator fee
  Submission sequence:
    Step 1: ERC20.approve(registryAddress, depositStake) where depositStake >= submissionMinDeposit()
    Step 2: registry.addItem(itemURI, depositStake) with msg.value = arbitrationCost
  "Stake more than minimum" rationale: signals confidence; makes spam less attractive
  Challenge stake formula: challengeStake = item.stake × challengeStakeMultiplier / MULTIPLIER_DIVISOR
  Challenge msg.value: arbitrationCost (same as LGTCR)
  Note: do NOT write a shared addItem playbook — PGTCR addItem(_item, _deposit) differs from LGTCR addItem(_item); actual tx call belongs in each flavor file (Phase 3)

Appeal funding (shared, no subsection — identical in LGTCR and PGTCR):
  requiredForSide = appealCost + appealCost × feeStakeMultiplier / MULTIPLIER_DIVISOR
  Winner/loser/shared multiplier selection based on currentRuling
  Loser half-time rule: loser must fund before the halfway point of the appeal window

OUT OF SCOPE for this file:
