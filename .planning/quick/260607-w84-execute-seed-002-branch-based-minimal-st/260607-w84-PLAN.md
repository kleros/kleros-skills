  <action>
    Create .github/workflows/sync-master.yml implementing the full dev→master sync.

    Directory creation: mkdir -p .github/workflows before writing the file.

    The workflow must implement all of the following in order:

    TRIGGER (on):
      push.tags: ["*@v*.*.*", "v*"]
      workflow_dispatch: (no inputs needed — manual override for hotfixes)

    CONCURRENCY:
      group: sync-master
      cancel-in-progress: false   # never drop a running sync; queue instead

    JOB: sync (runs-on: ubuntu-latest)

    STEP 1 — Generate app token:
      uses: actions/create-github-app-token@v2
      id: app-token
      with:
        app-id: ${{ secrets.GH_APP_ID }}
        private-key: ${{ secrets.GH_APP_PRIVATE_KEY }}

    STEP 2 — Checkout dev (full history not needed):
      uses: actions/checkout@v4
      with:
        ref: dev
        token: ${{ steps.app-token.outputs.token }}
        fetch-depth: 1

    STEP 3 — Set up Node (for npm test + update-digests):
      uses: actions/setup-node@v4
      with:
        node-version: '20'

    STEP 4 — Quality gate (run from repo root):
      name: Quality gate — npm test
      run: npm test

    STEP 5 — Digest freshness gate:
      name: Quality gate — digest check
      run: |
        node scripts/update-digests.js
        if ! git diff --quiet .well-known/agent-skills/index.json; then
          echo "ERROR: index.json digests are stale. Run 'npm run update-digests' and commit."
          exit 1
        fi

    STEP 6 — Strip dev-only files:
      name: Strip dev-only files
      run: |
        git rm -r --quiet --ignore-unmatch \
          .planning \
          test \
          scripts \
          package.json \
          yarn.lock \
          .yarnrc.yml
        # Remove root feedback/handover md files (glob patterns)
        git ls-files | grep -E '^(.*_FEEDBACK.*|HANDOVER[^/]*)\.md$' | xargs --no-run-if-empty git rm --quiet
        echo "Strip complete. Remaining tracked files:"
        git ls-files | head -40

    STEP 7 — Configure git identity for the bot commit:
      name: Configure git identity
      run: |
        git config user.name "kleros-skills-bot[bot]"
        git config user.email "kleros-skills-bot[bot]@users.noreply.github.com"

    STEP 8 — Commit the stripped tree:
      name: Commit stripped tree to master
      run: |
        # If nothing changed after strip (master already up to date), skip push
        if git diff --cached --quiet && git diff --quiet; then
          echo "No changes after strip — master is already up to date."
          exit 0
        fi
        git commit -m "chore(master): sync from dev (strip dev-only files)" \
          --allow-empty-message || true
        git push origin HEAD:master --force-with-lease

    NOTE on force-push strategy: use --force-with-lease (not --force) so the push
    fails if someone manually pushed to master between when we checked out and when
    we push. This is safer than --force while still being non-linear (master is not
    a normal ancestor of dev — it's a derived branch).

    NOTE on APP_NAME placeholder: use "kleros-skills-bot[bot]" as the bot identity
    in this plan. The actual bot slug must match the registered GitHub App name.
    Document in a comment inside the YAML that the user must update this to match.

    Commit the workflow file to dev:

      git add .github/workflows/sync-master.yml

      git -c commit.gpgsign=false commit -m "$(cat <<'EOF'
ci: add sync-master GitHub Actions workflow (SEED-002)

Tag-triggered (skill @vX.Y.Z and repo vX tags) plus workflow_dispatch.
Runs npm test + digest freshness check before stripping .planning/, test/,
scripts/, package.json, and root feedback/handover md files from master.
Uses GitHub App token for the push (app-id/private-key secrets required).

Phase 2 (deferred): add master branch ruleset restricting direct pushes.

Co-Authored-By: Claude <noreply@anthropic.com>
EOF
)"

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| tag push → workflow trigger | Only annotated/lightweight tags matching patterns fire the sync |
| workflow → master push | App token minted per-run, auto-revoked at job end; bounded 1h leak window |
| strip step → master content | git rm operates on the checked-out dev tree; no file outside strip-list is deleted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-w84-01 | Tampering | master branch | mitigate | force-with-lease push fails if master was manually pushed between checkout and push; job exits non-zero |
| T-w84-02 | Elevation of Privilege | GitHub App secrets | mitigate | App scoped to Contents:R/W on this repo only; no org-wide permissions; secrets stored as repo secrets |
| T-w84-03 | Denial of Service | concurrent syncs | mitigate | concurrency.cancel-in-progress: false — queues, never drops a sync |
| T-w84-04 | Information Disclosure | dev-only files on master | mitigate | strip step removes .planning/, test/, scripts/, package.json, feedback md before push |
| T-w84-05 | Spoofing | bot commit identity | accept | Bot uses [bot] suffix in name; distinguishable from human commits; loop-guard note in workflow comments |
</threat_model>
