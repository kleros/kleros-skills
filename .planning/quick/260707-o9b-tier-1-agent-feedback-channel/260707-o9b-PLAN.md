<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AI agent -> public GitHub issue tracker | Agent-authored free text (task description, quoted skill section, "what happened") becomes a public GitHub issue the moment it's filed — via MCP tool, `gh issue create`, or the URL fallback a human completes. |
| feedback/SKILL.md content -> agent's tool selection | The skill only recommends a channel priority order (MCP / gh / URL); it cannot force which transport an agent actually uses. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-260707-o9b-01 | Information Disclosure | Agent-authored issue body (task/what-happened/quoted-section fields) | mitigate | feedback/SKILL.md's "Before you file" section explicitly warns not to paste secrets, private keys, or user PII, since issues are filed publicly — instruction-level mitigation is the only lever available for a Tier 1 (no server-side) design. |
| T-260707-o9b-02 | Spoofing | "Skill version / digest" field is agent-self-reported, unverified | accept | Low-value target — worst case a maintainer briefly investigates the wrong revision; the field is optional precisely because it can't be verified, and the digest is still cross-checkable against `.well-known/agent-skills/index.json` history if it matters. |
| T-260707-o9b-03 | Tampering | Issue template YAML / feedback SKILL.md content itself | accept | Both files are repo-tracked, go through the same PR + CI (`npm test`, digest-freshness gate) as every other change; no elevated trust path introduced. |
| T-260707-o9b-04 | Denial of Service | Repeated automated issue filing (e.g. a misbehaving agent looping) | accept | GitHub's own rate limiting applies; this is a maintainer-facing volume concern, not a security boundary — no PII or funds at risk, and it's explicitly out of scope for a Tier 1 zero-infrastructure design. |
| T-260707-o9b-05 | Elevation of Privilege | N/A — no code execution, no auth changes, no new API surface | accept | This deliverable is documentation + a static issue form; there is no runtime component to elevate privilege in. |
</threat_model>
