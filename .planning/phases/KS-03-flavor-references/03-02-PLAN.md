**## Submit evidence**

**## Fund appeal**
To fund an appeal on a PGTCR dispute: first retrieve the active arbitratorExtraData via `arbitrationParamsChanges(index)` (the index that was active when the dispute was created — available from GraphQL query 4C's `challenge.arbitrationSetting.index` field). Use this extraData — not a top-level registry call — when calling `appealCost` on the arbitrator.

For appeal funding algorithm and formula: `shared-deposits.md § Appeal funding (shared)`. Note that round state (amountPaid, hasPaid) is available via GraphQL query 4C (item → requests → rounds → contributions) as an alternative to direct contract reads.

**## Withdrawal flow**
Source: draft §8E-§8F lines 471-478. PGTCR-unique two-step withdrawal — the withdrawal period allows challengers to dispute removal during the window:
1. Call `startWithdrawItem(itemID)` — sets `withdrawingTimestamp` on the item.
2. Read `withdrawingPeriod` from the registry (onchain read).
3. Wait until `block.timestamp >= withdrawingTimestamp + withdrawingPeriod`.
4. Call `withdrawItem(itemID)` to finalize and recover the ERC20 stake.
WHY the delay: "Withdrawal is intentionally time-locked so challengers can dispute removal requests during the withdrawal window — this prevents stake recovery during active disputes."

**## Withdraw fees and rewards**
Source: draft §8G lines 479-480. Call `withdrawFeesAndRewards(beneficiary, itemID, challengeID, roundID)`. ABI: `function withdrawFeesAndRewards(address _beneficiary, bytes32 _itemID, uint120 _challengeID, uint256 _roundID) external`. Note: PGTCR uses `uint120 _challengeID` not `uint256 _requestID` (LGTCR). See shared-abi-fragments.md.

**## Admin actions**
Source: draft §8.1 lines 488-510. GOVERNOR-ONLY — always simulate and confirm with the governor before executing any admin action. Governor-only actions: `changeArbitrationParams(...)` (new arbitrator + extraData + metaEvidenceURI — subject to `arbitrationParamsCooldown`), economic params (submissionMinDeposit, challengeStakeMultiplier, etc.), period params (submissionPeriod, reinclusionPeriod, withdrawingPeriod, arbitrationParamsCooldown), `changeGovernor(newGovernor)`. Note: arbitration param changes are rate-limited by `arbitrationParamsCooldown` — cannot change more frequently than the cooldown period.

**## Deploy a new registry (factory)**
Source: draft §9 lines 512-540. Key differences from LGTCR factory: single MetaEvidence (no registration/clearing split), 4 periods array (submissionPeriod, reinclusionPeriod, withdrawingPeriod, arbitrationParamsCooldown), 3 stake multipliers. Numbered steps: (1) locate factory for target chain, (2) prepare single MetaEvidence JSON + upload to IPFS, (3) set constructor params (arbitrator, extraData, metaEvidenceURI, token address, submissionMinDeposit, 4 periods, 3 multipliers), (4) confirm with user, (5) simulate, (6) call factory deploy, (7) capture emitted registry address.
